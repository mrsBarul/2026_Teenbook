## Что я делал

Выбрал тему про мое пространство. Написал .md статьи в директорию WEB, разобрался с WikiData и SPARQL, построил онтологию из понятий, которые удалось найти в базе.

## Понятия и связи между ними

Я выделил 9 понятий, которые описывают пространство подростка.

- **Подросток** - ключевая сущность
- **Сделай сам** - подросток может сам обустроить свою комнату
- **Личное пространство** - святая святынь для любого из подростков
- **Хаос** - творческий беспорядок
- **Порядок** - дисциплинированный порядок!
- **Родитель** - ключевая сущность для подростка
- **Сосед по комнате** - сестра или брат с которыми подросток делит комнату
- **Уют** - чувство комфорта, порождаемое окружающим пространством
- **Комната** - место где живет и отдыхает подросток

## SPARQL-запрос

Использовал один запрос — на получение понятий и их описаний из WikiData:

```sparql
QUERY = """
SELECT ?item ?itemLabel ?description WHERE {
  VALUES ?item {
    wd:Q26384       # do it yourself (сделай сам)
    wd:Q26270533    # personal space (личное пространство)
    wd:Q1787424     # chaos (хаос)
    wd:Q12893838    # order (порядок)
    wd:Q7566        # parent (родитель)
    wd:Q14754454    # roommate (сосед по комнате)
    wd:Q180516      # room (комната)
    wd:Q1367353     # comfort (уют)
    wd:Q131774      # adolescence (подросток)
  }

  SERVICE wikibase:label { bd:serviceParam wikibase:language "ru,en". }

  OPTIONAL {
    ?item schema:description ?description .
    FILTER(LANG(?description) = "ru")
  }
}
"""

```

## Как шла работа

Зашел Wikidata и искал там понятия похожие на те темы, по которым писались статьи в папке WEB.

Сделал простую mermaid markdown (.mmd) схему. Оказалось довольно просто и удобно!

## Личные ощущения

Больше всего времени ушло не на код, а на то, чтобы найти правильные понятия в WikiData. База большая, но социальная тематика там покрыта слабо — многие очевидные связи просто отсутствуют.
