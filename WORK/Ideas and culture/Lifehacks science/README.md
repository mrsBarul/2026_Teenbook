### 2. Схема связей между темами

В ходе работы была построена онтология, отражающая связи между пятью основными подтемами:

1. **Почему телефон греется** (физика) – связана с законом Джоуля-Ленца, теплопроводностью, работой процессора и аккумулятора.
2. **Газировка и кости** (химия + биология) – показывает влияние фосфорной кислоты на обмен кальция и здоровье костей.
3. **Почему мы видим сны** (биология) – описывает механизмы сна, роль REM-фазы, активность мозга (миндалевидное тело, гиппокамп).
4. **Как работают читы в играх** (программирование) – объясняет изменение кода и памяти, использование чит-кодов и трейнеров.
5. **Наука в быту** (повседневные явления) – охватывает кипение, трение, электричество, работу бытовых приборов.

**Горизонтальные связи:**
- Тепловые явления объединяют нагрев телефона и кипение воды.
- Электричество связывает аккумуляторы, батарейки и работу электроприборов.
- Информатика (память компьютера) имеет аналогию с памятью человека (гиппокамп).
- Химические процессы (фосфорная кислота) влияют на биологические структуры (кости).

Общая схема онтологии представлена в файле `images/Общая схема.png`. Детализированные схемы по каждой подтеме также сохранены в папке `images`:

### 3. Примеры SPARQL-запросов

Для получения данных из Wikidata использовались запросы, подобные приведенному в файле `scripts/4 читы.py`. Ниже пример запроса, который извлекает информацию о читах и связанных с ними играх:

```sparql
SELECT ?cheat ?cheatLabel ?cheatDescription ?game ?gameLabel WHERE {
  ?cheat wdt:P31/wdt:P279* wd:Q264609.
  OPTIONAL {
    ?cheat ?prop ?game.
    ?game wdt:P31/wdt:P279* wd:Q7889.
  }
  OPTIONAL { ?cheat schema:description ?cheatDescription FILTER(LANG(?cheatDescription) = "ru") }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "ru". }
}
LIMIT 50
```

### 4. Процесс работы

1. **Подготовительный этап**: изучение структуры Wikidata, подбор правильных идентификаторов (Q-номеров) для понятий, связанных с темами. Составление чернового списка понятий.

2. **Извлечение данных**: написание и выполнение SPARQL-запросов через сервис Wikidata Query Service. Для каждого понятия собирались основные свойства, описания, изображения. Данные экспортировались в JSON и помещались в `data/`.

3. **Построение онтологии**: на основе полученных данных и дополнительных знаний построены иерархические и горизонтальные связи. Схемы созданы с помощью инструмента Mermaid и экспортированы как PNG-изображения в папку `images/`.

4. **Формирование списка статей**: на основе онтологии составлен файл `concepts.json`, содержащий 20 ключевых понятий, по которым будут написаны статьи.

5. **Генерация текстов**: для каждого понятия с помощью языковой модели DeepSeek (промпт: *«Объясни для десятилетнего ребёнка простыми словами, с примерами и интересными фактами»*) были созданы статьи в формате Markdown. Статьи сохранены в папку `WEB/Ideas and culture/Lifehacks science/concepts/`.

6. **Автоматическая расстановка перекрёстных ссылок**: разработан скрипт на Python (с использованием библиотеки `pymorphy2` для учёта падежей), который находил в текстах статей вхождения названий других понятий и заменял их на markdown-ссылки вида `[понятие](файл.md)`. Скрипт также защищал уже существующие ссылки от повторной обработки. Код скрипта находится в `scripts/link_cross_references.py`.

7. **Оформление отчёта**: подготовлен данный файл README.md с описанием всех этапов и ссылками на результаты.

### 5. Личные ощущения от работы

Работа оказалась очень увлекательной и познавательной. Я впервые столкнулся с Wikidata и SPARQL – это мощный инструмент для получения структурированных знаний. Было интересно наблюдать, как из абстрактных запросов рождаются конкретные данные, которые затем ложатся в основу статей.

Генерация текстов с помощью ИИ тоже оставила яркие впечатления. Модель хорошо справлялась с задачей адаптации сложных научных понятий для детской аудитории, добавляла примеры и аналогии. Некоторые статьи пришлось немного корректировать вручную, но в целом результат получился качественным.

Наибольшую сложность вызвала расстановка перекрёстных ссылок с учётом падежей. Библиотека `pymorphy2` значительно упростила эту задачу, но для многословных терминов пришлось реализовать комбинаторный перебор форм. Тем не менее, итоговый скрипт работает корректно и связал все статьи в единую сеть знаний.

В целом, лабораторная работа позволила на практике применить методы искусственного интеллекта и семантических технологий. Результатом я доволен: получилась небольшая, но целостная энциклопедия для детей.
